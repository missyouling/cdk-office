# 内存优化指南

本文档介绍了如何在资源受限的环境中优化CDK应用的内存使用，特别针对2C4G配置的VPS环境。

## 优化原则

1. **连接池优化**：减少数据库和Redis的连接数
2. **并发控制**：限制同时处理的请求数量
3. **日志优化**：减少日志输出以降低内存占用
4. **缓存策略**：合理设置缓存大小和过期时间

## 配置优化详解

### 数据库连接池优化

```yaml
database:
  max_idle_conn: 5     # 空闲连接数从10减少到5
  max_open_conn: 20    # 最大连接数从100减少到20
  conn_max_lifetime: 300  # 连接最大生命周期从3600秒减少到300秒
```

### Redis连接池优化

```yaml
redis:
  pool_size: 20        # 连接池大小从100减少到20
  min_idle_conn: 5     # 最小空闲连接数从10减少到5
```

### 工作进程优化

```yaml
worker:
  concurrency: 10      # 工作进程并发数从50减少到10
```

### 日志优化

```yaml
log:
  level: "warn"        # 日志级别从info调整为warn
  max_size: 50         # 日志文件最大大小从100MB减少到50MB
  max_age: 7           # 日志保留天数从30天减少到7天
  max_backups: 3       # 日志备份数量从10个减少到3个
```

## 监控和调优

### 内存使用监控

建议使用以下命令监控应用的内存使用情况：

```bash
# 监控进程内存使用
watch -n 1 'ps aux | grep cdk-server | grep -v grep'

# 查看系统内存使用情况
free -h

# 查看Go程序的内存统计
curl http://localhost:8000/api/v1/debug/stats
```

### 性能调优建议

1. **根据实际负载调整参数**：
   - 观察应用在实际使用中的连接数需求
   - 根据并发用户数调整连接池大小

2. **使用系统级优化**：
   - 启用swap空间以防止OOM
   - 调整Linux内核参数优化内存管理

3. **定期重启服务**：
   - 定期重启服务以释放内存碎片
   - 使用crontab设置定期重启任务

## 故障排除

### 内存不足错误

如果出现内存不足错误，请检查：

1. 是否有内存泄漏
2. 连接池配置是否合理
3. 是否需要进一步降低并发数

### 连接池耗尽

如果出现连接池耗尽错误，请：

1. 检查数据库和Redis的连接数限制
2. 适当增加连接池大小
3. 检查是否有连接泄漏

## 进一步优化建议

1. **使用轻量级组件**：
   - 考虑使用更轻量级的Web框架
   - 替换重量级依赖库

2. **启用Go的内存优化参数**：
   ```bash
   GOGC=20 ./cdk-server api
   ```

3. **使用pprof进行性能分析**：
   ```bash
   go tool pprof http://localhost:8000/debug/pprof/heap
   ```
